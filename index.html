解释黑箱模型

Posted on 2018-09-30 | Edited on 2018-10-08

当我们做一个模型的时候，除了追求准确率之外，也希望模型是可解释的，例如这个人为什么是坏人。但是主流的准确率较高的机器学习算法都是黑箱模型，变量对模型和结果的影响不如回归之类的统计学模型那么显而易见。在本文中我整理了几种常用的，我觉得比较make sense，也比较容易理解的解释黑箱模型的方法。

首先，我们得明确，所有model interpretation都是基于fixed model，即已经train好的model，任何需要retrain model得出variable贡献的方法都是不合适的，因为我们想要知道的是在这个train好的model里这个variable的作用。

我将这些方法和常用的术语都列在这儿，可以比较清晰地看出这些方法不同的侧重点。

variable importance

变量对form model overall的贡献。

variable contribution

变量对模型prediction overall的影响程度

Partial dependence plots & Individual conditional expectation (ICE) plots

保持其他所有变量不变的情况下，变化target variable，看prediction的变化。ICE plots对每个observation画一条线，partial dependence plot是一个summary level的图，可以理解为average了ICE plots。

LIME

Local interpretable model-agnostic explanations (LIME) 是看单个observation的prediction的值受哪些变量的影响。

从上面的简介可以看出，我们可以将解释黑箱模型的方法分为几类：

方法 度量对象 作用对象 结果类型
variable importance form model overall numeric value
variable contribution prediction overall numeric value
Partial dependence plots & ICE plots prediction overall/individual plot
LIME prediction individual numeric value

团伙交易欺诈模型实践

Posted on 2018-06-07 | Edited on 2018-09-30

交易欺诈侦测模型需要在海量正常交易中找到有问题的交易。以往，模型建立在会员维度，即预测一笔交易的发起人是否有欺诈风险，训练集中一条记录代表一个会员。由于欺诈行为有一定的隐蔽性，欺诈会员的行为和属性与正常用户有一定的相似性，且正负样本的比例可以失衡到1:10000，这样的模型产生的false alarm可能在比例上很低，但从false alarm的数量上来说，仍是一个不小的数字。对一家小型支付公司而言，没有那么多的人力去review这些量的false alarm。因此，业务上对反欺诈模型的精度要求很高，同时对覆盖率也有较高的要求。
